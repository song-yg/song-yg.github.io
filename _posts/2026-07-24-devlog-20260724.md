## 오늘 진행된 것 정리

### 1. LLM 연결 문제 진단 마무리
- 모델 변경(nvidia/nemotron) 후 한때 성공했다가 다시 429 재발 → **근본 원인 확정**: OpenRouter 무료 티어 "하루 요청 수 한도"(계정 단위, 크레딧 없으면 하루 50건) — 파이프라인 1회 실행이 이미 60건 안팎을 쓰기 때문에 모델을 뭘로 바꿔도 소용없었다는 결론
- **결정**: 발표 전까지 결제/승인 안 하기로 함 → LLM은 죽은 채로 발표까지 감

### 2. "LLM 100% 실패 상태에서도 안 죽는지" 검증
- 실제 실행 로그로 확인: 관련성 필터/재분류/그룹핑/요약 전부 429로 전멸해도 **크래시 없이 완주**, 저장·이메일까지 정상 → 지난번 크래시(파일 동기화 문제)도 해결 확인

### 3. 키워드 버그 2건 수정
- `축산물 수입관세` → `축산물 할당관세`: 실제 기사에 안 쓰이는 표현이라 검색 0건이었음 (naver_collector.py + keyword_tagger.py 사전 둘 다 수정)
- `축산물 수급` 노이즈 원인 규명: 네이버 검색이 공백 구분 키워드를 AND로 처리해서 생기는 오탐 → `_phrase_present()` 문구 인접성 필터 신규 구현, 시트 재활성화

### 4. "지난주 대비 증감" 기능 신규 구현
- `scorer.py`(국내/해외 판별 로직 단일화), `storage.py`(`previous_week_dir`), `category_aggregator.py`(`compare_with_last_week`), `main.py`/`deploy.py`(콘솔·이메일·summary.md 3곳에 반영) — 2주 연속 실행 시나리오로 종단 테스트까지 완료

### 5. WATTAgNet 재조사 → 영구 제외 확정
- 직접 로컬 테스트 스크립트 실행 → **캐시 문제가 아니라 Cloudflare 봇 차단**임을 스크린샷으로 확인
- 우회 시도 안 하기로 결정, 관련 코드 주석 전부 정정(SITES, SINGLE_PAGE_ONLY_SITES 등)

### 6. 5.2 정부기관 보도자료 매칭 — 취소 결정
- 문서(원본/요약본) 반영, 목차/미해결 목록 정리

### 7. 코드 주석 전면 최신화 (가장 큰 작업)
- 13개 파일(`gdelt_collector.py` 1191줄 포함) 전부에서 "2026-07-XX 추가", "담당자 지적", "실측 확인됨" 같은 진행 과정 서술 제거 → 현재 설계 이유만 남김
- 파일마다 `py_compile` + 자체 테스트로 회귀 없음 확인
- 테스트 파일 2개(`test_timeline_timing.py`, `test_timelinevol_only.py`)는 안 쓴다고 하셔서 정리 대상에서 제외
