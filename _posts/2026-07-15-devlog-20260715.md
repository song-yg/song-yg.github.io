## 3차 LLM 보조

오늘 원래 목표였던 작업이다.

임베딩에서 근사치가 애매한(borderline) 기사 쌍만

LLM에게 "같은 사건이 맞는지" 물어보도록 구현했다.

<br>

```python
# issue_grouper.py
def stage3_llm_assist(borderline_pairs: list[tuple[dict, dict, float]]) -> list[tuple[dict, dict, float]]:
    """
    2차에서 애매 구간에 걸린 쌍들을 LLM에 물어봐서, "같은 사건"으로 확정된 쌍만 골라 반환.
    """
    if not borderline_pairs:
        return []

    key_env_var = "OPENROUTER_API_KEY" if LLM_PROVIDER == "openrouter" else "ANTHROPIC_API_KEY"
    api_key = os.environ.get(key_env_var)
    if not api_key:
        print(f"[issue_grouper] {key_env_var} 없음(LLM_PROVIDER={LLM_PROVIDER}) - 3차 LLM 보조 생략, "
              f"애매 구간 {len(borderline_pairs)}쌍 전부 '안 묶음' 기본값 유지")
        return []

    confirmed = []
    for i in range(0, len(borderline_pairs), LLM_BATCH_SIZE):
        batch = borderline_pairs[i:i + LLM_BATCH_SIZE]
        results = _call_llm(batch, api_key)
        if results is None:
            continue  # 이 배치만 안 묶음 유지, 다음 배치는 계속 시도
        for (a, b, sim), same_event in zip(batch, results):
            if same_event:
                confirmed.append((a, b, sim))
    return confirmed
```

## 이슈 그룹핑 main.py 연결

여기서 의외의 함정이 있었다.

국내/해외를 먼저 나누면 국가가 다른 동일 이슈는 영원히 만날 수 없었다.

결국 그룹핑을 먼저 하고, 국내/해외 분리는 나중에 하도록 순서를 바꿨다.

```python
# main.py
def score(articles: list[dict], model, top_n: int = 5) -> tuple[list[dict], list[dict]]:
    # ① 전체 기사를 대상으로 group_issues()를 한 번 호출
    groups = issue_grouper.group_issues(articles, model=model)

    # ② 그 결과 그룹들을 국내/해외 축으로 "나눠서" scorer에 넘긴다.
    domestic_groups = []
    international_groups = []
    for group in groups:
        domestic_part = [a for a in group if a.get("source") == "네이버"]
        international_part = [a for a in group if a.get("source") != "네이버"]
        if domestic_part:
            domestic_groups.append(domestic_part)
        if international_part:
            international_groups.append(international_part)

    domestic_ranked = scorer.score_and_rank(domestic_groups, top_n=top_n)
    international_ranked = scorer.score_and_rank(international_groups, top_n=top_n)
    return domestic_ranked, international_ranked
```

## 오늘 버그를 두 개 잡았다

### 첫 번째

FALSE_POSITIVE_FILTERS.

분명 만들어 놨다.

그런데 실제로는 한 번도 동작한 적이 없었다.

GDELT 제목이 예상했던 문자열이 아니라 구두점 앞에 공백이 들어간 형태로 반환되고 있었던 것이다.

```python
# gdelt_collector.py
_SPACE_BEFORE_PUNCT = _re.compile(r"\s+([,.;:!?])")


def _normalize_spacing(text: str) -> str:
    """구두점 앞의 공백을 제거해 비교용으로 정규화한다 (예: "hand , foot" -> "hand, foot")."""
    return _SPACE_BEFORE_PUNCT.sub(r"\1", text)


def _is_false_positive(keyword: str, title: str) -> bool:
    patterns = FALSE_POSITIVE_FILTERS.get(keyword, [])
    if not patterns or not title:
        return False
    title_normalized = _normalize_spacing(title.lower())
    return any(_normalize_spacing(p.lower()) in title_normalized for p in patterns)
```

결국 문자열을 계속 추가하는 대신 비교 전에 공백을 정규화하도록 수정했다.

class _FakeEmbeddingModel:
    def encode(self, texts: list[str], normalize_embeddings: bool = True):
        import numpy as np
        rng = np.random.default_rng(seed=42)
        dim = len(texts) + 1
        vectors = []
        next_free_axis = 1  # 0번은 곡물/사료 그룹 전용으로 예약
        for text in texts:
            vec = np.zeros(dim)
            if "옥수수" in text or "grain" in text.lower() or "feed price" in text.lower():
                vec[0] = 1.0
            else:
                vec[next_free_axis] = 1.0

            vec += rng.normal(scale=0.02, size=dim)
            vectors.append(vec)
        return np.array(vectors)
