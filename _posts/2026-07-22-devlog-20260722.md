### WATT_collector.py

```python
ORDINAL_PATTERN = re.compile(r"(\d+)(st|nd|rd|th)")
```

r"..."(raw string) → 이스케이프 무력화.

\d+ → 숫자(digit) 하나 이상. \d는 0~9 중 하나, +는 "1개 이상 반복". 그래서 1, 21, 31 같은 숫자를 통째로 잡아냄.

(st\|nd\|rd\|th) → st, nd, rd, th 중 하나라도 걸리면 잡아냄. \|는 "또는(OR)"을 의미.

( ) 괄호 두 개 → 각각 캡처 그룹(capture group). 매치된 부분 중에서 숫자와 서수 접미사를 따로따로 꺼내 쓸 수 있게 나눔.

<br>

```python
preview = (r["body"] or "")[:100]
```

r["body"] or "" → 만약 body가 None/빈 값이면 빈 문자열 ""로 대체 (에러 방지용 안전장치). or는 앞 값이 false(None, 빈 문자열 등)면 뒤 값을 씀.

<br>

<br>

```python
page.goto(url, timeout=30000, wait_until="networkidle")
```

page.goto(url) → 브라우저로 해당 URL 방문.

timeout=30000 → 30000밀리초(=30초) 안에 안 뜨면 타임아웃 에러.

wait_until="networkidle" → "네트워크 요청이 잠잠해질 때까지" 기다림. 요즘 사이트는 JS로 나중에 데이터를 더 불러오는 경우가 많은데, 그런 추가 로딩까지 어느 정도 기다려주는 옵션.

<br>

```python
try:
    page.wait_for_selector("h5 a, h4 a", timeout=15000)
except Exception:
    print(f"[watt] 목록 로딩 실패 또는 타임아웃: {url}")
    return []
```

page.wait_for_selector("h5 a, h4 a", ...) → CSS 셀렉터로 "h5 태그 안의 a 태그" 또는 "h4 태그 안의 a 태그"가 화면에 나타날 때까지 최대 15초 기다림.

(,는 CSS 셀렉터에서 "또는"을 의미) (networkidle만으로는 부족할 때를 위한 이중 안전장치)

15초 안에도 안 뜨면 Exception 발생 → except로 잡아서 에러 메시지 로그 찍고 빈 리스트 반환.

<br>

```python
if title and link and not any(pat in link for pat in EXCLUDED_PATH_PATTERNS):
```

title and link → 제목과 링크가 둘 다 비어있지 않아야 함 (빈 문자열은 falsy).

any(pat in link for pat in EXCLUDED_PATH_PATTERNS) → 제너레이터 표현식(generator expression). EXCLUDED_PATH_PATTERNS라는 리스트를 하나씩 돌면서, 그 패턴이 link 안에 포함되는지 확인. 하나라도 포함되면 any(...)는 True.

셋 다 만족해야 진짜 기사 링크로 인정하고 results에 딕셔너리로 추가.
