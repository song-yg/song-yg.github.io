## 오늘 진행된 것 정리

### 1. 코드 전체 안정성/효율성 점검 및 보완
- **세션 재사용**: `naver_collector`, `relevance_filter`, `issue_grouper`(3차), `llm_summarizer` 4개 모듈에 `requests.Session()` 적용
- **naver_collector.py**: 페이지네이션 중 예외 발생 시 이미 모은 결과가 통째로 날아가던 버그 수정 (부분 실패 시 데이터 보존)
- **WATT_collector.py**: `page.goto()` 무방비 상태였던 걸 try/except로 감싸 부분 실패 방지, `browser.close()` try/finally 보장
- **keyword_source.py**: 프로세스 내 CSV 캐싱 추가 (ko/en 호출 시 중복 요청 제거)
- **relevance_filter.py**: WATT 소스는 LLM 호출 없이 자동 통과 (업계 전문지 특성상 오매칭 구조적으로 불가능)
- **issue_grouper.py**: 3차 LLM 보조에 id 기반 부분 복구 이식 (relevance_filter와 동일 개선)
- **storage.py**: 파일 쓰기 실패를 개별적으로 흡수, 부분 성공 지원

### 2. Claude Code 핸드오프 프롬프트 작성 + 알고리즘 문서(v15) 지속 업데이트

### 3. 실전 실행 → 버그 다발 발견 및 수정
- **git push 충돌**(non-fast-forward) 발견 → `git pull --rebase` + `concurrency` 설정 추가로 해결, 실전 검증 완료
- **CSV URL 오설정**(`pubhtml`) 발견 → 원인 규명(코드 문제 아님, 리포 설정 문제)
- **naver_collector 하드코딩 fallback**에 `구제역`(위험 키워드) 잔존 발견 → v5 활성 키워드로 동기화, gdelt fallback도 동일 조치
- **keyword_source CSV 타임아웃**(15초→30초) 완화
- **FALSE_POSITIVE_FILTERS 키 불일치 버그** 발견(`"foot and mouth disease"` vs 실제 키워드 `"foot-and-mouth disease"`) → 오탐 필터가 무력화돼 있던 것 확인 후 수정

### 4. 국내/해외 재분류 버그 — 발견 → 재발견 → 근본 수정
- `scorer.py`에 넣었던 한글 재분류 로직이 **실제로는 안 쓰이는 죽은 코드**였던 걸 실전에서 재확인
- `main.py`의 진짜 실행 경로에 직접 반영, 재현 테스트로 확정
- **GDELT `language` 필드** 발견(raw.json 직접 열어보다 발견) → 글자 세기 방식보다 신뢰도 높은 이 필드를 우선 사용하도록 개선(`_is_korean_gdelt_article`)

### 5. 최종 목표(카테고리별 Top N) 설계 및 전체 연결
- 국내/해외 축과 독립적인 카테고리 축 설계, `CATEGORY_TOP_N` 상수화(현재 1)
- `scorer.score_by_category()`(다수결 카테고리, 기타 제외, 빈 카테고리 생략) 구현
- LLM 요약 + `storage.py`(scored.json/summary.md)까지 엔드투엔드 연결, 통합 테스트 통과
- **실전 첫 성공** — 국내 6개, 해외 3개 카테고리에서 정상 출력 확인

### 6. 유사도 디버그 아티팩트 구현
- `export_similarity_scores()` — 그룹핑 임계값 튜닝용 CSV, GitHub Actions 아티팩트로 업로드(git 커밋은 안 함)
- 실전 업로드 성공 확인(17,837쌍)

### 7. OpenRouter 무료 티어 근본 문제 확인
- 새 API 키를 넣어도 429가 전혀 안 풀리는 것 확인 → `openrouter/free`가 **계정별 한도가 아니라 전 세계 공용 풀**이라 키 재발급으로는 해결 안 됨을 규명
- `OPENROUTER_MODEL`을 구체 모델명으로 바꾸는 방안 제안 + GitHub Variables 설정 위치 안내

### 8. 그룹핑 임계값/프롬프트 최종 조정
- `THRESHOLD 0.75→0.7`, `BORDERLINE_MARGIN 0.05→0.06` (담당자가 디버그 CSV 보고 결정)
- LLM 프롬프트에 **경남/제주 등 국내 지역 구분 예시**, **단신 vs 종합기사 구분 규칙** 추가
- **연쇄(체이닝) 병합 버그** 발견 — A~B, B~C 확정만으로 A~C도 안 물어보고 셋이 통째로 묶이던 구조 확인 → 클리크(완전 그래프) 검증 로직으로 수정, 단위 테스트 3종 통과

### 9. GDELT 학습형 스킵 키워드** (다른 세션에서 진행)
- `SKIP_KEYWORDS`(수동) 옆에 학습형 목록 신설 — 실패 2회 연속 확인 시 자동 등재
- `state/gdelt_skip_keywords.json`로 영속화, `run-pipline.yml`에 `state/`도 커밋 대상으로 반영 완료
